Where do I even start? I started this website in 2022, and it has grown up with me since then. It is truly my greatest achievement ever. And I really love this site. I love the weird writing from older days, and I love exploring the archive to see what nonsense I was getting up to. The development is always fun and I am so glad to have this website represent me to the public.

The modern website is built on SvelteKit, with a custom blog system that I will detail later. It has a glossary system, a search, and tags and other forms of organization. I host it on a DigitalOcean Ubuntu droplet, using Nginx as the reverse proxy. I use Cloudflare as my domain registrar and I use their other services as well.

So, let's start by telling a history of this website.

History

Before I start, quickly, I will be using the "new" repo and old repo as reference, along with some older blog posts. I will also use my college essay as reference. So lets start with the original website.

This was the first entry. Grandma, you might have recognized this. This entry set into motion a passion for coding and writing that I still have.

000webhost

000webhost is where things really started to kick off. It was a fully static site host, similar to GitHub pages. I learned subpages, index.html files, and STYLING! Although, if you had looked at the styling back then, you'd have rather me not style it.

I started adding some little toys and tools to the site, like a mailto link generator, some contact info and an about page, and most importantly, the Wordle Solution Generator. At this point, this was my magnum opus. Some of my family members reached out, saying it was very cool! This was shocking to me. It isn't possible anymore, as wordle stores answers secretly now, but it used to work. I was so proud of it then- not really anymore. The light of god does not shine anywhere near that code.

GitHub pages

The GitHub pages website is what I consider the beginning of the golden age of this website. It was hosted on cheespeasa1234.github.io, and was a static site as well.

Later in this era, I got better at other forms of programming, especially Java, and I started writing more for the website. This is also when I started making LeviSchedule, which would become my greatest achievement (for a bit).

At this point, I was using most features of static site development, and I began to hit my limit. I began having thoughts of moving to self hosting, and I began research. This transition would take a very long time, but it started here.

Old natelevison.com

When I first began hosting on my server, I used express.js and a static site folder, so same as before. I began improving the website, and really tried to make it my own. I added a blog around this time, and this is the era of my website that lasted the longest.

I began by "revamping" the visuals (they still looked awful) and improving the code. I reorganized some pages and added some new ones, and began developing a brand new blog system. I was starting to write more about my life, and needed a new place for a blog.

This is also when my grandma revealed to me she had been reading my website all the time. This was awesome to hear, and is the primary plot point of my personal college essay. I began putting "thanks for reading, Grandma" or something similar at the end of all my articles.

When my blog system finally went up, I uploaded frequently, and started to neglect the updates page, which was its own thing now. It is now only for significant updates to the website.

Modern-day natelevison.com

After some time, the blog system began to grow some cobwebs. It was using string replacing on the backend, and some nasty stuff to work "statically". It was absolutely ABHORRENT code. So I rewrote the whole website in SvelteKit.

I also began to redesign the site a final time using Bootstrap, and some other tools. I really enjoyed redesigning it, and I am quite proud of how it looks now! I consolidated some pages and redesigned the blog system from scratch, and I really like how it works.

This will serve as a quick explanation before I write a full article on it, but the blog uses xml files which store data about the article and loads them in using a slug and an API endpoint. I made the search code better and it sorts better. I also remade the glossarizer and it is a lot simpler now.

Skils

Obviously, the primary skill I have gained in this experience of making this website is web development. I learned how to use CSS, HTML, and JS, and the current state of this website is pretty much exactly how much I know.

I learned about version management and documentation, as I would frequently take month-long hiatuses and version control helped me get back on track. I also learned about hosting and GitHub, which has been a toolbelt item ever since.

I learned about how the internet works, like IPs, DNS, domains, hosting, HTTP, and more, and I learned about hosting, using Ubuntu, services, servers, clients, express.js, Deno, SvelteKit, and more.

Conclusion

This website holds a special place in my heart. It was the place I wrote about dealing with the passing of my Grandma, it's where I went to deal with social isolation, and it shows my learning process and growth so well. This website is like a part of my mind now.

I am not done with this website; and I hope I will never be. I love what it has become and I have set the groundwork for a dense repository of all of my writings. Expect more articles soon. Thank you for reading, Grandma.
